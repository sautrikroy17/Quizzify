# 🔧 Backend — C++ OOP Core Engine

This folder contains the **C++ backend engine** for Quizzify AI, demonstrating advanced Object-Oriented Programming (OOP) concepts applied to the quiz platform architecture.

## 📁 Files

| File | Purpose |
|------|---------|
| `QuizzifyCore.cpp` | Full C++ OOP implementation of the Quizzify engine |

## 🧠 OOP Concepts Demonstrated

| Concept | Where Used |
|---------|------------|
| **Encapsulation** | `Question` class — private fields, public getters |
| **Abstraction** | `IMeasurable` interface (pure virtual), `Assessment` abstract class |
| **Inheritance** | `PracticeQuiz` & `CompetitiveTest` extend `Assessment`; `Student` & `Admin` extend `User` |
| **Polymorphism** | `conductAssessment()` overridden in derived classes; virtual dispatch via `User*` array |
| **Smart Pointers** | `unique_ptr<Assessment>` — modern C++ memory management, no raw `new`/`delete` |
| **Design Patterns** | Singleton concept in `main()` engine; Template Method pattern in Assessment |

## 🔨 How to Compile & Run

```bash
g++ -std=c++17 -o QuizzifyCore QuizzifyCore.cpp
./QuizzifyCore
```

## 🏗️ Class Architecture

```
IMeasurable (interface)
    └── Assessment (abstract)
            ├── PracticeQuiz
            └── CompetitiveTest

User (base)
    ├── Student
    └── Admin
```

## 🔗 Relationship to Frontend

The C++ engine models the **same domain logic** used by the Next.js frontend:
- `Question` → mirrors quiz questions generated by Gemini AI
- `Assessment` → mirrors quiz sessions tracked in PostgreSQL
- `Student` → mirrors the `User` model in Prisma schema
- `Admin` → mirrors administrative controls
- Leaderboard score logic mirrors `totalScore` & `quizzesTaken` in the DB
